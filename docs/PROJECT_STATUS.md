# TaskMg プロジェクト状況ドキュメント

**最終更新**: 2025-12-30

## 📝 プロダクト概要
**TaskMg** は、目標達成とタスク管理を統合した先進的な Web アプリケーションです。
**「目標（Target）から逆算して、日々の行動（Task）を設計する」** というコンセプトの元、単なるToDoリストではなく、OKR、WOOP、バックワード・デザインといった複数のフレームワークをシームレスに切り替えて利用できることが特徴です。

### 主な機能
- **高度なタスク管理**: プロジェクト、ラベル、時間帯（Time Block）、所要時間による多角的なフィルタリングと統計表示。
- **目標設計ウィザード**: 対話形式で目標を明確化し、行動計画へと落とし込むガイド機能。
- **マルチフレームワーク**: OKR、WOOP、バックワード・デザインなど、状況に応じた目標管理手法をサポート。
- **フレームワーク Wiki**: 各手法のベストプラクティスをアプリ内で学習・参照可能。

## 🎯 次のステップ

1. **パフォーマンス最適化 (優先度：高)**
   - **データ購読の高度化 (重要ボトルネック)**:
     - `store-raw.ts` での Firestore `snapshot` 処理時に、前回データとの比較（Deep Compare または JSON比較）を行い、変更がない場合は配列参照を維持する仕組みを導入。
     - これにより `useTasks` および `useProjects` の配列参照を安定化させ、アプリケーション全体(`App.tsx`)の不要な再レンダリングを抑制する。
   - **CustomFilterList の最適化**: 保存済みフィルターごとの件数計算 (`getProcessedTasks` 呼び出し) が重いため、結果のメモ化またはデバウンス処理を導入。
   - **コンポーネントの最適化**: `TaskItem` および `ProjectItem` の `React.memo` 化を行い、レンダリングコストを最小化。
   - **仮想スクロール**: TaskList のタスク数増大に備え、`react-window` 等の導入を検討。

2. **設計のクリーンアップ (優先度：中)**
   - **App.tsx のスリム化**:
     - サイドバー定義を `components/sidebar/SidebarContent.tsx` 等に切り出し。
     - DnD ロジック (`handleDragEnd`) をカスタムフックまたはロジックファイルへ分離。
   - **コード共通化**: 日付フォーマット処理や UI 密度設定ロジックを `utils` に統一。
   - **設定の永続化**: ソート等の表示設定をストア (`settings-store`) と連携。

3. **残務処理・品質向上**
   - 未使用ユーティリティ関数の洗い出しと削除。
   - テストカバレッジの拡充。

4. **新機能開発への復帰**
   - 既存機能のブラッシュアップや、ユーザーフィードバックに基づく新機能追加。
